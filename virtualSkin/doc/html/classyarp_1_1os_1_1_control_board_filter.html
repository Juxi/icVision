Detailed Description

A proxy for a YARP device driver that can also filter data.

A YARP device driver has three ports:

  • /targetName/state:o
  • /targetName/command:i
  • /targetName/rpc:i

This class creates three more ports to provide a transparent interface to the device driver: 

 * 		- /filterName/state:o
 *		- /filterName/command:i
 *		- /filterName/rpc:i
 *
 *

The filter provides a mechanism to cut the connection (effectively cutting off the user from the YARP device driver) as well as inject data into the exchange between user and device driver.

Note:
Gregor's original implementation has been modified such that when the filter is cut, it still allows the state port to transmit data
